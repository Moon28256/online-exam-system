# 🚀 在线考试系统 - 部署说明

> 目标：把本地运行的项目通过一个公网网址暴露出去，发给别人即可使用，数据库连通不出错。

## 一、部署架构

```
别人浏览器  ──公网网址──>  cpolar 隧道  ──>  本机 8080 (Spring Boot jar)
                                          │
                                          ├── 前端静态资源 (打包进 jar 的 static/)
                                          └── API (/user/**, /question/** ...)
                                                │
                                                └── MySQL (exam_system)
```

- **单端口**：后端 jar 同时托管前端 + API，所以只需穿透一个端口 **8080**。
- **hash 路由**：前端用 `createWebHashHistory`，刷新任意 `#/...` 路径都不会 404。
- **数据库连接**通过环境变量注入，无需改代码。

## 二、本地准备

### 1. 数据库
确保 MySQL 已运行，并建好库与表：
```bash
mysql -u root -p
mysql> CREATE DATABASE IF NOT EXISTS exam_system DEFAULT CHARACTER SET utf8mb4;
mysql> USE exam_system;
mysql> source sql/schema.sql;
mysql> source sql/seed.sql;
```

### 2. 一键构建
```bash
# Windows
deploy\build.bat
# Linux / macOS
bash deploy/build.sh
```
脚本会依次：构建前端 → 拷贝 `dist/*` 到 `exam-backend/.../resources/static/` → 打 jar。
成功后 jar 位于：`exam-backend/exam-backend/build/libs/exam-backend-0.0.1-SNAPSHOT.jar`

### 3. 本地启动验证
```bash
deploy\run.bat        # 或 bash deploy/run.sh
```
浏览器访问 **http://localhost:8080**，能登录、考试即说明本地一切正常，再进行下一步穿透。

> `run.bat` 顶部有 `DB_URL` / `DB_USER` / `DB_PASSWORD`，部署到他人电脑或改用云端数据库时，改这几行即可，**不要改代码**。

## 三、用 cpolar 内网穿透（免费方案）

cpolar 提供免费公网域名，无需服务器、无需备案，适合课程演示。

### 1. 安装 cpolar
- Windows：到 https://www.cpolar.com/download 下载安装包，注册账号后在「认证」页面复制自己的 **Authtoken**。
- 执行一次认证（只需一次）：
  ```
  cpolar authtoken <你的token>
  ```

### 2. 开启隧道，映射本地 8080
```bash
cpolar http 8080
```
终端会输出一条形如 `https://xxxxxx.r2.cpolar.cn` 的公网网址——**这就是发给别人的网址**。

### 3. 对外访问
- 保持上面两个窗口开着（jar 在 run 窗口跑、cpolar 在另一个窗口维持隧道）。
- 把 `https://xxxxxx.r2.cpolar.cn` 发给别人，浏览器打开即可登录使用。
- 由于后端 CORS 已 `allowedOriginPatterns("*")` 且同源部署，前端请求 API 不会再有跨域问题。

### 4. 可选：固定域名 / 开机自启
- 免费版域名每次重启会变。如需固定二级子域名，需 cpolar 付费保留。
- 也可把 cpolar 隧道写成后台服务（cpolar 官网有「守护进程」教程），实现开机自启。

## 四、常见问题

**Q1：别人访问提示数据库连接失败？**
A：本机的 MySQL 必须保持运行；`run.bat` 里的 `DB_PASSWORD` 要与实际 MySQL 密码一致。数据库跑在你本机，公网用户经你后端访问，**他们不需要装 MySQL**。

**Q2：访问 `https://...cpolar.cn` 后白屏 / 刷新 404？**
A：前端用 hash 路由，正常不会 404。若白屏，F12 看 Console：多为后端没起或端口不对。确认 `run.bat` 输出 `Started ... on port(s) 8080`。

**Q3：登录返回 401？**
A：token 过期或未登录。重新登录即可；后端 `WebConfig` 已放行 `/`、`/index.html`、`/assets/**` 等静态资源，不影响登录。

**Q4：免费 cpolar 有流量/带宽限制？**
A：免费版有连接数与带宽限制，课程演示足够；正式长期使用建议买保留域名或上云服务器。

**Q5：想部署到云服务器？**
A：把仓库 clone 到服务器 → 装 JDK17 + MySQL + Node → 跑 `deploy/build.sh` + `deploy/run.sh`，然后用 Nginx 反代 8080，或直接开放 8080 端口。无需 cpolar。
